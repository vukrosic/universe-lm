# Taste — 066 label-smoothing

## r1 — 2026-06-11 — verdict: reject

- **Closed-by-data sibling in same family.** 010-polyloss ran NULL at tiny1m3m on 2026-06-09 (Δ −0.0053 vs ctrl 6.5991, inside the 0.0059 ctrl-to-ctrl gap — `closed.md` + `010-polyloss/evidence.md`). 010's own idea.md explicitly states: *"PolyLoss and LabelSmooth are not mathematically identical… but at tiny1m3m the val-loss delta vs plain CE is unlikely to be large enough to separate the two empirically."* The simplest soft-target lever is the empirical twin of the polynomial-correction lever we already burned a slot on; the predicted outcome is another null.
- **Already wired in-tree.** `configs/output_head_ablations.py` already contains a `LabelSmooth` config and `training/trainer.py` references it — label smoothing is a *baseline* the loss-side aux family was scaffolded around, not a new bet. The idea is "flip the pre-existing flag", which is exactly the "tweak everyone has already tried" failure mode in `idea-taste.md`.
- **Portfolio crowded.** 067-confidence-penalty (entropy bonus on token distribution) is queued right next to this one — the entropy form of the same "soften the output" lever. Plus 010-polyloss already ran. Three output-side regularizers stacked in a row is `revise` (diversify) territory even if each were individually fine; combined with closed-by-data, it's a `reject`.
- **Info-free outcome.** Miner's own framing — *"If tiny1m3m is overconfident, label smoothing is the simplest baseline to reveal it"* — concedes that the bet only fires if the output head peaks. 010's null is direct evidence the head **isn't peaking hard enough** at 3M tokens on a 0.94M model for soft-target levers to clear the variance bar. A second null on the same family teaches nothing new; a sub-noise positive would be inconclusive (and per `idea-taste.md`, "inconclusive" is itself a taste failure).
- **No magnitude prediction.** Pitch lacks a sharp "we expect Δ = X because Y at *this* tier" — the citation is generic Müller-2019 LM-scale evidence, not a mechanism that argues the lever fires at 3M tokens specifically where the polynomial cousin already nulled.
