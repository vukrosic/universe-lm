# Taste log — 036 LAMB

## r1 — 2026-06-11 — verdict: reject
- **Tier mismatch — the only failure that matters.** LAMB's mechanism is a *large-batch* lever: trust-ratio scaling `||w||/||u||` exists to keep per-step updates from overshooting when batch size is 8k-32k+ (the paper's headline is BS 32,868 BERT in 76 minutes). At tiny1m3m we run ~92 total steps at small batch on 3M tokens; there is no large-batch update-magnitude problem for the trust ratio to fix. The `transfer-risk: low` self-tag is wrong in the direction that matters: it transfers *to large-scale BERT pretraining*, not *down to a 0.94M model trained for 92 steps*. Reject per the niche-fit rule: "An idea whose payoff only appears at larger scale is a `reject`."
- **No mechanistic argument for tiny1m3m.** The "why it's worth a slot" is "good baseline for whether our tiny model wants trust ratios more than AdamW's raw adaptive moments" — a vibe, not a bet. At small batch with a global LR schedule, the trust ratio collapses to roughly a constant per-layer rescale of the AdamW step that the LR already controls. No sharp "we expect X because Y at the tiny1m3m scale" exists in the pitch.
- **Crowded optimizer family.** Active queue already carries 031 adam-mini, 032 adams, 033 sophia, 034 adan, 035 agc, 037 lookahead, 038 swan, 039 apollo, 040 adafactor — all at `needs-taste`. The taste guide explicitly says "the 5th optimizer-momentum variant in a row is a `revise` even if each is individually fine"; LAMB sits inside that wave with the weakest tiny-scale story of the lot (its sibling AGC is the same trust-ratio idea applied as a clip — cheaper and arguably more relevant here).
- **Information value of a null is low.** A tiny1m3m null tells us "trust ratios don't help at small batch" — already the consensus reading of the LAMB paper. Not worth a slot.
- **Closes cleanly against prior closures.** Aligns with the same logic that rejected 018-ademamix ("bet cannot fire at tiny1m3m") and 008/012-gated-deltanet ("off-niche on tier"). LAMB is the optimizer-family analog.
