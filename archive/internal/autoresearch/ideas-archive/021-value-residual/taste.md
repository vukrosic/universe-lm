## r1 — 2026-06-09 — verdict: accept
- **Mechanism**: clean. Compute V at layer 0 once per forward (`V_1 = W_V^0 · x`), stash it, and in every later block mix it into that block's freshly-projected V: `V_l = (1 − λ_l)·V_l + λ_l·V_1`, with one learnable scalar `λ_l ∈ ℝ` per block, init 0 → step-0 reduces to the baseline exactly. The cross-layer value shortcut gives later blocks a direct, gated gradient path back to the input's value representation, bypassing the residual-stream-and-reproject round trip. < 25 LoC: stash `V_1` in the forward, add one `nn.Parameter` of shape `(n_blocks,)` to the model, one blend line before `attn @ V` or before `o_proj`'s input.
- **Transferability**: distinct from every closed axis. Closed `V/Q/K/O embeds + combos, q_gain / k_gain` is about *value embedding axis scaling* (operates on the projection input); 021 is a *value stream shortcut* (operates on the projected V across layers). The closed `attn sink` lever addresses a different facet of the same underlying attention-concentration problem. None of the 015/016/017/020 batch touches cross-layer value stream — 015 is optimizer, 016 is per-head logit bounding, 017 is per-sublayer output bounding, 020 is post-softmax mass decay. 021 is a *new site* (inter-block value stream) and a *new function* (cross-layer mixing, not local bounding). The paper (Zhou et al. 2024, ResFormer/SVA) shows consistent wins at 7B+; tiny1m3m has 6 layers and narrow heads, so leverage is bounded but plausible — attention concentration collapse with few heads is the named bet, and a single learnable scalar per block is exactly the right shape for the question.
- **Identity-init**: cleanest possible — `λ_l = 0` makes `V_l = V_l`, so step-0 is bit-identical to the unmodified baseline. No "close to identity" approximation, no init-LR subtlety, no schedule dependence. The A/B isolates the *blend* from the *baseline* with zero risk of step-0 interference.
- **Bet** (crisp, in the miner's own words, sharpened): "we expect a small but real val-loss drop because attention concentration in 6 layers × few heads makes a single sink token dominate V aggregation, and a learnable V-shortcut from layer 0 gives later blocks a way to read the input's value stream without paying the residual-reproject tax. λ_l > 0 means the model wants the shortcut; λ_l ≈ 0 after training means the question is closed at this scale." The second clause is what makes the null informative: a per-block λ_l readout post-run is itself a finding.
- **Null teaching**: a clean null at tiny1m3m closes a specific question — "cross-layer V mixing's benefit is concentrated in deeper stacks (≥12+ layers) where residual drift between blocks is large; at 6 layers, the per-block V path is short enough that the round trip is cheap." This partitions the depth-stability axis from 016/017/020 cleanly: 016/017/020 test whether *local* stability (per-head / per-sublayer / per-head-mass) is the binding constraint at 6 layers; 021 tests whether *inter-block* value propagation is the binding constraint. All four results together map the depth-stability landscape.
- **Portfolio**: 015, 016, 017, 020 are in the active batch plus 020 in flight. 021 is a *categorically different site* (cross-layer value stream, not per-head/per-sublayer/per-head-mass). The active batch is depth-stability-heavy (016/017/020/021 = 4 of 5), but each tests a *different site* and *different function* — running them all is the right portfolio move because the results partition the depth-stability axis into four sites. Not a redundancy concern.
- **Caveats for the reviewer**: (1) set a low-to-moderate PASS bar (~-0.005 to -0.01); the bet is at the small end of the paper's reported effect. (2) The experiment naturally partitions the per-block readout — record the post-train `λ_l` values per block and append to evidence.md; a uniform `λ_l → 0` after training is a stronger null than "inside variance". (3) Decide and pin one canonical form: a *single global* `λ` shared across blocks is a weaker, more conservative lever; the *per-block* `λ_l` form is the paper's full lever. The miner's spec uses per-block, which is the right choice — confirm in the plan. (4) The definition gate should explicitly check that the layer-0 V is stashed from the *post-LN* input (i.e., the same `x` that gets Q/K/V-projected at layer 0) so the shortcut is at the right semantic site. Wire as a `use_value_residual` flag on `TransformerBlock` (or model) with init 0.
