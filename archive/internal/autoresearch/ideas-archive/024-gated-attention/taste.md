# Taste log — 024 gated-attention

## r1 — 2026-06-10 — verdict: accept

- **Mechanism (clean).** Per-head scalar gate applied to the head output `o_h = A_h V_h` *after* AV and *before* the output projection that mixes heads: `o_h ← o_h · sigmoid(W_g^h · x_t)`, with `W_g^h : d_model → H` (per-head scalar variant) — the cheap, correct shape for tiny1m3m's narrow heads. ~30 LoC, one `nn.Linear(d_model, H)`, one sigmoid, one elementwise multiply inside `MHA.forward`. Fires every step (no EMA, no schedule). Parameter-cheap: `H·d_model` weights per layer (~2K at 6L·8H·256d), no extra activations of consequence.
- **Distinct site (the leverage test).** Closed axes hit the score function (NSA / diff-attn / hybrid heads), the mask (SWA / dilated attn), the position encoding (RoPE / NoPE / FIRE), the logit range (logit-softcap, QK-Norm 016), the per-sublayer output (sub-LN 017), the attention *probabilities* post-softmax (FoX 020), the cross-layer *value stream* (V-residual 021), the *softmax itself* (softpick 022), the *residual stream pre-attention* (canon-conv 023), and the *logit temperature* (SSMax 025). None of these gates the head *output value* `o_h`. The Qiu et al. 2025 framing — non-linearity injection between AV and o_proj, head-suppression sparsity, attention-sink-free — names a site the closed set never touched. Qwen-style "gated attention" blocks name the same site. This is a *new* lever, not a 5th-of-its-family.
- **Identity-init (cleanable, with a known caveat for the reviser).** The miner's "sigmoid(0) = 0.5, fold into o_proj init" is a *convention* — the gate is constant 0.5 at step 0, so step-0 `o_h` is scaled by 0.5× and o_proj must be initialized at 2× to recover the baseline magnitude. That works but the plan must **explicitly pin which form is used** — the reviser should pick one of: (a) 2× `o_proj` init when `use_gated_attn=True`, (b) use `2·sigmoid(W·x + b)` so the constant is 1, or (c) use a *residual* form `1 + sigmoid(W·x + b)` (constant 1 at W=0) which is the cleanest step-0 ≡ baseline. This is a definition-gate call, not a taste block — but flagging it now saves a round trip.
- **Bet (sharp, falsifiable).** "Gate injects input-conditional non-linearity and sparsity between attention and o_proj, letting each head suppress its own output when uninformative, killing attention sinks and massive activations → expect a val-loss drop on tiny1m3m." The second clause is the null: if a clean null, head-output gating is redundant with SwiGLU's gating at our scale, and the sink pathology doesn't bind at 0.94M / 6L — partitions a specific question distinct from 020's "multiplicative mass decay is sub-threshold" and 016's "per-head logit bounding is sub-threshold."
- **Information value either way.** Win: a sub-30-LoC structural lever stackable on FIRE, sub-2K params per layer, no schedule, no HP. Null: closes "is head-output gating binding at tiny1m3m?" with a clean A/B. Both outcomes are worth logging.
- **Niche fit.** Mechanism (not HP), identity-init-able (modulo the reviser's choice above), runs at tiny1m3m, no data/infra dependency, no tier-mismatch (unlike 018 AdEMAMix's 7k-step EMA half-life that broke the 92-step run). The paper validates at 1.5B+; at 0.94M the lever may null — but that's the result we want logged, not a taste problem.
- **Portfolio.** Active queue has 020 (forgetting decay), 021 (V-residual), 022 (softpick), 023 (canon-conv), 025 (SSMax) — six attention-side levers, which is *crowded*, but each tests a *different site* (A-prob decay / cross-layer V / softmax swap / residual conv / logit temperature / head-output gate). 024 fills the only site the active set doesn't already test. No redundancy concern at the *site* level. The depth-stability map the active batch is building is genuinely the right portfolio move — 024 belongs in it.
- **Wire as a single `use_gated_attn: bool = False` flag on MHA**, threaded through `TransformerBlock` and `LLMConfig`. Trt config class adds the flag to the chosen baseline (definition gate picks FIRE-equipped or non-FIRE — recommend FIRE-equipped so the A/B partitions the *orthogonal* head-output axis from 009's additive position bias, matching the 020 ctrl convention).

This earns a slot. The bet is sharp, the A/B is clean, the null is informative, the lever is distinct from every closed axis and every active lever, and the niche fits cleanly.
