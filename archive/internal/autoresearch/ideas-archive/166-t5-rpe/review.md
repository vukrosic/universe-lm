# Review log — 166-t5-rpe

## r1 — 2026-06-14 — verdict: approve
- **Source is real and current.** Raffel et al. JMLR 2020 (arXiv:1910.10683) is the canonical T5 paper, JMLR vol. 21 no. 140. BigBird (Zaheer et al. 2020), REALM (Guu et al. 2020), and LongT5 (Guo et al. 2021) all re-use the same 32-bucket RPE on attention logits. Authors and citations line up.
- **Mechanism is a mechanism.** Additive logit bias indexed by bucketed relative position `floor(log2(|i−j|+1))` with a learned per-head per-bucket scalar. Init is `zeros(H, B)`, so the bias is exactly 0 at step 0 and the logit-add branch is `scores + 0 = scores` — bit-identical to the un-biasd baseline. Distinct from the *Q/K rotation* family (RoPE, FIRE) and from 152 (per-head constant bias — no positional component, mathematically nulled by softmax). T5-RPE is **not** a constant per head, so the 152 cancellation caveat does not apply.
- **tiny1m3m only.** Spec names the tier and seed-42 explicitly; no `screen20m`, no ladder.
- **Not a duplicate of a closed axis.** Cross-checked `autoresearch/closed.md`: the closed PE axes (NoPE, RoPE base sweep, 009-FIRE-PE WIN, 013-CoPE drift) are all *rotational* or *content-aware*; 152 (per-head constant bias) and 155 (per-head τ) are non-positional. The *additive-logit-bias, position-indexed* family is unclosed at our tier for autoregressive LMs. T5-RPE is the canonical, well-validated entry on that family.
- **LoC budget.** Spec estimates ~30 LoC, plus the standard `LLMConfig` flag + `Tiny1M3MConfig` subclass (~15 LoC) + `models/llm.py` pass-through (~5 LoC). Total ~50 LoC, well under the 200 ceiling. Pattern matches 152/155: register the `nn.Parameter` inside the `if use_t5_rpe:` block so the off-path RNG state is preserved.
- **Falsifiable.** Plan must pin (a) the control and (b) a pass-bar number. Control candidates: the cached tiny1m3m baseline (mean 6.4346 ± 0.0458 after the 162 fresh re-cache, per the runner cache) **or** the NoPE baseline — depends on the lever mode picked below. Box noise ±0.01 val loss at tiny1m3m means a real effect should clear ~−0.02. Null is informative (closes the additive-bias PE family at our tier).
- **Transfer-risk tag is honest.** `med` matches the scale evidence (T5-XXL 11B + 100M+ in BigBird/REALM/LongT5) and the encoder-decoder-vs-decoder-only caveat. The spec calls this out explicitly. Win carries, null is a transferable negative (rotational family binds at the 135M recipe too).

**Findings for the plan (not blocking — implementer picks):**
- **Lever mode pin.** The design sketch offers two: (A) REPLACE RoPE with T5-RPE, control = NoPE/null baseline; (B) ADD on top of current RoPE+FIRE, control = 009 FIRE+RoPE baseline. The spec's "starting as 'added on top of un-rotated Q/K' gives the cleanest step-0 identity" hints at (A). Recommend (A) — it tests T5-RPE in isolation against a known-clean control, and the rotational vs. additive axis is the actual bet. If (B) is picked, the control must be 009-FIRE-PE with FIRE on (not the bare baseline). Plan must commit to one.
- **Bucket utilization at T ≤ 2048.** The log2-bucket function yields 12 distinct bucket indices for `|i−j| ∈ [0, 2047]` (bucket 0 = dist 0, bucket 1 = dist 1-2, …, bucket 11 = dist 1025-2048). Buckets 12..31 are zero-initialized and unused at our T. T5 used 32 buckets because T5 has 512 input length. The plan should call this out as intentional (no change needed) and document the actually-used bucket count.
- **Manual-path trigger.** The 152 pattern requires the new flag to be added to the manual-attention-path trigger list at `models/layers.py:2601-2615` so SDPA's flash/efficient backends don't change step-0 numerics. Plan must include this.
- **Step-0 byte-identity self-check.** The standard pattern: `torch.manual_seed(42)`, build control and test, compare logits at step 0 within fp32 noise. Plan should include this as the self-check gate before launching the run.

**Verdict: approve.** Sound mechanism, real source, falsifiable, distinct from closed axes, identity-init-able, ~50 LoC, mode and bar pin are plan-level choices the implementer can make cleanly. Round reset to 1.
