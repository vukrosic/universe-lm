# Taste log — 196 block-residual-ema

## r1 — 2026-06-15 — verdict: reject
- **Direct empirical prior against this exact structure: 164-q-carry null Δ=+0.0360 wrong-sign at 3.6× the plan bar.** 164's mechanism is *Q-side cross-block residual mixing with α_l init 0 + .detach()* — additively blending a previous block's contribution (with no gradient flow) into the next block's residual stream, gated by a learnable scalar starting at 0. 196's mechanism is structurally the same: full-residual cross-block carry with per-block EMA coefficient β_b init 0, `out_{b-1}.detach()` to keep gradients local. The only deltas are (a) "full residual" vs Q-side, (b) per-block β_b vs per-layer α_l, (c) EMA framing vs scalar gate. None of these deltas is a mechanistic argument for *why the more-general lever would win where the simpler one lost* — they are knob-shape changes, not axis changes. 164 was null at 0.94M *with no horizon-scaling issue* (α_l is fully active at step 0, like 196's β_b). The "weight-EMA tier-mismatch doesn't apply here" rebuttal is correct but irrelevant — the relevant counter is 164, not 110/132/134.
- **Cross-block residual-mixing axis is closed at 0.94M.** Beyond 164: 150-xlayer-feedback rejected at round-3 cap ("cross-block attention path structurally hostile at d_model=64/12L — cross-attn output dominates residual stream at any nonzero effective gate"), 116-hyper-connections null (mHC residual-split not amortized at 6L), 017-sub-ln-sandwich null (depth-drift lever doesn't accumulate at 6L). 021-value-residual is the lone WIN (Δ=-0.034) but it is *V-only with a tighter design* (learnable scalar α init 0, detach, narrow carry on a single projection) — not a generalized residual-stream EMA. A 196 NULL adds a 4th data point to a closed axis. A 196 WIN would have to clear the 0.005 pass-bar *and* overcome the wrong-sign direction 164 established; even a sub-bar |Δ|<0.01 "win" is structurally uninformative (it would be noise around a null axis).
- **The "residual stream EMA is novel" framing is correct but the bet is dull.** Yes, residual-stream EMA is not weight-EMA, and yes, no published LM evidence. But novelty ≠ leverage: 196's bet is "if win → residual-stream smoothing is a binding lever that weight-EMA isn't." That sentence is fine; the issue is the prior probability. With 164, 150, 116, 017 all closing variants of cross-block residual mixing at 0.94M (three wrong-sign / null, one narrow WIN), the prior on 196 winning is well below 0.5 — and the wrong-sign direction in 164 actively argues against a win. The bet would need to be "we expect Δ > +0.005 (or even a positive Δ) because the EMA framing gives the model a *different* lever than the closed 164 axis" — but the framing doesn't give it a different lever, it gives it a different knob on the same axis.
- **Information value of a null is the deal-breaker.** Per the niche's "single A/B must teach us something whether it wins or loses" rule: a 196 NULL would join 164 (Q-side) + 150 (cross-attn) + 116 (mHC) + 017 (sub-LN) + 111 (DropPath) as a 5th closing-null on the cross-block-residual / depth-stability family at 0.94M. The EMA framing doesn't add a new finding — it's the same axis with a different label. A 196 WIN at Δ<-0.005 would be a real surprise and worth carrying to 135M. The win/null information asymmetry argues for a `reject` here: most likely null, null is uninformative, win has to be substantial. The single seed constraint amplifies this — even a "real" win at |Δ|=0.006 is one-seed noise; we'd need substantial separation to believe it.
- **Leverage is small (+0.001% of 0.94M) but that's not the killer; the family is.** 12 scalar β coefficients with sigmoid init ≈0 is a tiny parameter budget, but the *activation-level* effect is what matters and is bounded by the per-block contribution magnitude — at d_model=64, attn+ffn output norms are small, so even β=0.5 only blends in a 0.5× magnitude nudge from the previous block. The expected Δ under any reasonable β schedule is |Δ|<0.01, inside the null band.
- **The niche-fit bits are OK, which is what makes this a hard call.** Mechanism (not HP), zero-init-able via sigmoid(β_raw) init -10, no horizon-scaling problem, bit-identical at step 0 — all the niche checks pass. The reject is on **prior probability + information value**, not on shape. The bar is "worth a slot" not "well-shaped"; a well-shaped bet on a closed axis is still a no.
- **Not a `revise` because the gap is structural, not sharpenable.** "Sharpen the bet" doesn't help when the issue is that the closed axis is the relevant prior. "Different EMA from weight-EMA" — already argued, doesn't help. "Frame the post-null info value relative to 164" — would land as "5th null in the family," which is uninformative. "Tighten the pass-bar" — would just make a likely-null more likely to be classified null. The lever would need a different mechanism (e.g. asymmetric: only blend the EMA at depth >N, or only on specific blocks) to escape the 164 prior — and that's a re-pitch, not a taste-time revise.
