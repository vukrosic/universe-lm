# Review log — 192 pre-rope-qk-rotation

## r1 — 2026-06-15 — verdict: approve

- **Source real and current**: RoFormer/RoPE (Su et al., arXiv:2104.09864, Neurocomputing 2024) is real and verified; 154-rebased-attn (WIN), 185-static-per-head-k-rotation (rejected, recode-cap), 172-per-head-rope-base (closed null) are all in-repo real and correctly cross-referenced. Wang et al. "ResFormer / Residual Rotation" (2024) cited as adjacent — the lever doesn't depend critically on it, but the idea should verify the citation string before plan (cursory check: arXiv:2310.18836 or similar; if not pinned, fine — it's flavor, not the lever). No fabricated citation at the core.
- **Mechanism, not HP**: learned per-head, per-pair static 2D rotation on **both** Q and K, applied **before** RoPE's position-dependent rotation. Init at φ=0 ⇒ R=I ⇒ step-0 byte-identical to baseline exactly (cos(0)=1, sin(0)=0 are exact in IEEE-754 fp32). The implementer must verify `max_abs_diff(trt_step0_logits, ctrl_step0_logits) == 0.0` in smoke. Architectural lever, not LR/schedule/init-constant.
- **tiny1m3m only**: every reference is tiny1m3m (0.94M · 3M tok, seed 42); cached baseline reference `5b8a7fea8963` (val 6.3988 ±0.04). No screen20m or larger-tier leakage. ✅
- **Not already closed**: distinct from
  - 154-rebased-attn (WIN, fixed *random shared* rebase on K and V **pre-softmax**) — 192 is *learned, per-head, Q+K, pre-RoPE*. Different placement relative to position (154 is position-agnostic pre-softmax; 192 is position-independent pre-position-mix), different tensors (K,V vs Q,K), different mechanism (random fixed vs learned).
  - 185-static-per-head-k-rotation (rejected, exhausted 3 recode rounds, axis abandoned) — 185 is *post-RoPE K-only*. 192 is *pre-RoPE Q+K*. Different tensor (K vs Q+K) and different placement (post vs pre). The recode-cap close of 185 is on the *post-position placement*, not the rotation axis itself; 192 explicitly tests the *pre-position placement* and so is the cleanest opposite-arm of the same two-armed bet. Not a mathematical duplicate of 185.
  - 172-per-head-rope-base (closed null, per-head *position-dependent* RoPE base frequency) — 192 is *position-independent*. Different lever shape (static basis change vs angular-frequency modulation). The two together bound "where is the orthogonal-rebase axis binding — pre-RoPE static re-orient vs in-RoPE frequency specialization".
  - Per-head scalar family (152/155/160/166, all closed null) — 192 is per-head *matrices* on Q/K (8 plane pairs × 4 heads × 12 blocks = 384 params), not per-head scalars on logits. Two orders of magnitude more params per head, two orders of magnitude more degrees of freedom. Different axis.
  - 176-v-pre-av-norm (closed null) — different tensor (V, pre-attention-output) and different op (norm vs rotation). Genuinely fresh lever.
- **Implementable in < 200 LoC**: same footprint as 185. New subclass `Tiny1M3MPreRoPEQKRotationConfig(Tiny1M3MConfig)` with `use_pre_rope_rotation: bool = True` + `pre_rope_rotation_init: float = 0.0`; one `nn.Parameter(torch.zeros(n_heads, d_k // 2))` per attention block, applied **before** the existing RoPE call. Apply per-pair 2D rotation via reshape + small matmul on plane pairs (`[B, T, H, d_k//2, 2]` → rotate last dim). Build block-diagonal R_h implicitly (no full d_k×d_k materialization needed — same trick as 185). Param budget: +384 params (+0.041% of 0.94M), negligible. ~30–60 LoC across `models/layers.py`, `configs/llm_config.py`, `models/llm.py`. No file outside the 154/185 footprint.
- **Falsifiable pass/fail bar**: clear WIN/NULL/DRIFT bins tied to the cached control `5b8a7fea8963` (mean 6.3988, band 0.04). WIN = trt ≤ ctrl − 0.005 with two-ctrl bracket clearance; NULL = |trt − ctrl| < 0.01; DRIFT = trt > ctrl + 0.01. Sub-noise effects → inconclusive per one-seed-only rule, never "add seeds to confirm". Standard bar.
- **transfer-risk justified**: `med` is correct. The lever *form* (learned pre-RoPE static rotation) is novel at any published scale; the underlying orthogonal-rebase mechanism is well-validated by 154's WIN at 0.94M and RoPE-family wins at LLaMA-1/2/3 / Mistral / Qwen (≥7B). Static rotation is structurally independent of sequence length (φ is a per-head basis change, position-agnostic), so transfer is plausible. Tag matches the citation.
- **Information value — high in all three outcomes, sharp two-sided bet**:
  - WIN → pre-RoPE static re-orient binds; the orthogonal-rebase axis is binding on the *position-agnostic* side, not just the post-position side. Unlocks a clean Phase-2 lever at ≥135M.
  - NULL → the axis is specifically the *post-position* placement (i.e., the rotation needs to interact with position mix to be useful); pre-RoPE nulls because the rotation is washed out by RoPE's position-dependent mix anyway. Closes pre-RoPE cleanly.
  - DRIFT → learned basis change is *worse* than identity — strong negative, rules out the whole pre-RoPE rotation family.
  No "meh" branch.
