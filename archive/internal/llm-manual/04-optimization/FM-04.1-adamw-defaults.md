# FM-04.1 — AdamW with β=(0.9, 0.95), wd 0.1, grad-clip 1.0 is the default

**Rule.** Pre-train with **AdamW**: `β₁=0.9, β₂=0.95, ε=1e-8`, **weight decay 0.1** (decoupled,
*not* applied to norms/biases), **gradient clipping at norm 1.0**. These are the LLaMA/nanoGPT-
lineage defaults and a safe starting point across scales.

**Confidence.** **[E] Established.** This exact configuration recurs across essentially every
open frontier model and the scaling-law literature.

**Scope.** Dense transformer pre-training. Note **β₂=0.95 is pre-training-specific** — the
generic/fine-tuning default is β₂=0.999. The lower β₂ makes the 2nd-moment estimate adapt
faster, improving large-scale stability.

## Evidence / details

- **Why AdamW, not Adam:** decoupled weight decay (regularize weights without polluting the
  adaptive 2nd-moment estimate). Standard since Loshchilov & Hutter 2019.
- **β₂=0.95 vs 0.999:** pre-training uses the faster-adapting 0.95; fine-tuning reverts to
  0.999 (or 0.95). Multiple recent works converge on 0.95 for pre-training.
- **Weight decay 0.1**, applied to weight matrices only — **exclude LayerNorm/RMSNorm gains
  and biases** (a standard, easy-to-miss implementation detail). For fine-tuning, ~0.01.
- **Grad clip 1.0 by global norm** is near-universal and pairs with the above for stability.
- **Practitioner's law:** *a well-tuned schedule + AdamW beats a poorly-tuned exotic
  optimizer.* Spend tuning effort on LR/schedule (FM-04.2) before swapping optimizers.
- Newer optimizers (Lion, Sophia, Muon, Shampoo/SOAP) show gains in specific settings but
  none has displaced AdamW as the *default*. Muon in particular is gaining traction in the
  small-model speedrun community.

## Reproduced by us?

**Held fixed, by rule.** Per RULE 0 the ledger does not mine the optimizer; these defaults are
the fixed background for structural experiments. The one time we touched LR/momentum
(`tiny-update-starvation` draft) the axis was closed afterward. We use these settings; we
don't claim them.

## Falsifier

For the *default* claim: an optimizer config that beats this one, paired and clearing noise,
*reproducibly across labs and scales* would displace it. (Single-setting wins don't — they're
the existing C-tier alternatives.)
