# Taste log — 201 MLP Token Mixer on Attention Output

## r1 — 2026-06-15 — verdict: revise
- **Mechanism is muddled — commit to one formulation, drop the other two.** The pitch presents three different "MLP token mixer" shapes and explicitly hedges between them: (a) "W_1, W_2 ∈ R^{d_model × d_model} applied per-channel across tokens" — that's a *channel mixer* applied per-token, not a token mixer (the operation is the standard FFN up/down-proj, no T-axis mixing happens); (b) "1D depthwise Conv1d with kernel=T or k=4" — the author correctly notes this is just 163 with a different kernel, so it loses to 163 on the same slot; (c) "per-channel linear across T: W_1: [d_model, T], W_2: [T, d_model]" — at T=2048 this is 2×64×2048 = 262,144 params per block × 12 blocks = 3,145,728 params (+335% of 0.94M) — infeasible at tiny1m3m. The pitch cannot land on all three. The re-pitch must pick (c) — the only shape that is *actually* a token-mixing MLP — and bring the param cost down to a tractable, fair number (see next bullet).
- **+10.4% param cost (+98k) is unfair-by-param — same failure mode as 070-mtp-head taste-reject.** The pitch's headline "Params: 2 × d_model × d_model = 8192 + 1 α × 12 blocks = 12 α's, total ~98k extra params (+10.4% of 0.94M)" is the d_model×d_model channel-mixer shape (formulation a), not a token mixer. 8,192 weights per block at 12 blocks is 98,304 — that is just a per-token FFN on the attention output, which (a) is not a token mixer, (b) gives the optimizer +10% free capacity to spend wherever it likes, and (c) makes any null result uninterpretable ("did the lever not bind, or did the model just absorb the capacity into something else?"). Per 070-mtp-head taste-reject: "param-budget injection makes A/B unfair-by-param; info-value zero — null reconfirms paper, win would be unbelievable on one seed." A re-pitch that wants to be a *token-mixing* A/B has to use formulation (c) and pay the per-channel-across-T cost honestly, then bring it down via one of: (i) per-block stochastic — only 4 of 12 blocks have the mixer (33% mixing, ~26k/block per the mixed blocks), (ii) low-rank factor across T: `W_1: [d_model, T, r]` with r=4 ⇒ 64×2048×4×2 = 1,048,576 params per block, still too many — try r=2 or downsample T (e.g., 256-token chunks), (iii) commit to a kernel=T depthwise conv (1 weight per channel) — that is 163 with full kernel and group=d_model, +64 weights × 12 blocks = 768 params (+0.08%), trivially fair, and the cleanest "global vs local" framing. Pick one and own the param math.
- **163 is in needs-implement — 201 is informationally conditional on 163's result.** The pitch's own claim is "201 is the MLP analog of 163 — broader token mixing (kernel = full token axis or large window) vs conv's narrow kernel. The bet: if the binding axis is *global* token mixing on attention output (not local), 201 wins where 163 doesn't." That bet is *only* informative if 163 has already run. If 163 nulls at k=3, then 201's "wider mixing helps more" prediction is in tension with 163's null (a wider kernel not binding implies 201 is also null), and the only way 201 wins on top of 163 null is "the post-attn mixing axis is global-by-default and k=3 misses the wider range" — a narrow win condition. If 163 wins at k=3, then 201 is the natural "does *more* mixing help" follow-up. Either way, the pitch's information value goes up sharply once 163's evidence.md is in. **Condition the re-pitch on 163's outcome.** If 163 has not run by the time the re-pitch is written, the miner should explicitly write "we will revise the bet once 163 lands" — that is honest and the re-pitch still gets to claim structural novelty for the global axis. If 163 already ran, the re-pitch must take 163's result as a premise and frame the A/B as the conditional follow-up.
- **gMLP / MLP-Mixer citations are loose — both replace attention, neither adds an MLP path alongside it.** Tolstikhin et al. (MLP-Mixer) and Liu et al. (gMLP) both *replace* the attention sublayer with an MLP token-mixing sublayer. 201's proposal is *additive* — keep attention, add an MLP token-mixing path on top. That is a structurally different lever with a much weaker mechanistic motivation: MLP-Mixer works because you don't have attention; here the model already has content-based token mixing (softmax) and the question is whether a parallel *position-based* token-mixing path adds anything the attention sublayer wasn't already providing. The re-pitch should drop the MLP-Mixer framing and commit to "gMLP-style spatial gating as a *complement* to attention" — and acknowledge that gMLP's spatial gating has no in-LM validation at 0.94M (gMLP was ImageNet-only; Liu et al. 2021 didn't test LMs). The transfer-risk tag is currently `med` — that's right, but the re-pitch must say so.
- **Niche fit / zero-init / step-0 bit-identity: clean.** The α=sigmoid(α_raw) with α_raw init -10 ⇒ α ≈ 4.5e-5 ⇒ bit-identical baseline at step 0 (after RNG-state discipline, raw-`Parameter` shape, no `nn.Linear` construction). That's solid. Identity-init-able ✓. Mechanism (not HP) ✓. No data/infra needed ✓. Plausibly shows effect at tiny1m3m if the cost framing is right. Transfer-risk: med is right. The lever shape is fine; the implementation needs to commit.

**Concrete revision path for round 2:**
1. Pick **one** mechanism. Recommendation: the per-channel-across-T linear (formulation c) implemented as a low-rank factor — `W_1: [d_model, T, r=4]`, `W_2: [T, r=4, d_model]` — or per-block stochastic (4/12 blocks have it). State the exact param count and the percentage of 0.94M. Or commit to a kernel=T depthwise conv (1 weight per channel, +64 weights × 12 blocks = 768 params, +0.08%) and reframe the bet as "global depthwise vs 163's local depthwise on the same post-attn axis."
2. Drop the MLP-Mixer / gMLP-as-replacement framing. Frame as "gMLP-style spatial gating added *alongside* attention, not as a replacement" — and acknowledge gMLP has no in-LM validation, transfer-risk stays `med`.
3. Condition the bet on 163's result. If 163 has not run, write "this lever becomes informative once 163 lands — if 163 wins, 201 tests whether *wider* mixing helps; if 163 nulls, 201 tests whether a *global* (not local) post-attn mixing axis is the binding constraint the k=3 conv missed." If 163 has run, take the result as a premise.
4. State the sharp bet in one sentence. Current pitch's bet is "MLP mixes globally vs conv's local kernel" — that's mechanism description, not a prediction. Rewrite as: "We expect 201 to be a clean null (≤|Δ|0.005) at 0.94M regardless of 163's result, because attention's softmax already provides global content-based token mixing and an additive global *position-based* path has no axis the optimizer can exploit at 0.94M/3M tokens" — or whatever the predicted Δval window is. The current pitch has no magnitude prediction.
5. The "intuition: content-based vs position-based mixing" line is the right one — keep it, but tie it to a *testable* prediction. The test is: does the optimizer ever pull α away from 0? If it does, by how much, and does that move val loss?
