The pitch's own framing ("a null would indicate the depth benefit only materialises at longer training horizons, informative for the 10M+ ladder") is the right null framing.

- **Fits the niche, low cost.** One new config dataclass (~30 LoC) in `configs/llm_config.py` mirroring the existing `Tiny1M3MConfig`; no model/forward-pass code change. Mechanism-level (config reallocation), not an HP. Identity/zero-init-able criterion is N/A for a config swap. Runs at tiny1m3m, seed 42.
