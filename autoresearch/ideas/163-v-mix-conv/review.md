- **Findings for the implementer (not blocking — proceed to plan with these pinned):**
  - **Conv weight MUST be constructed as a raw `nn.Parameter(zeros(d_model, 1, k))` with center-tap = 1.0 set inline — NOT as `nn.Conv1d(…)` followed by `.data` reassignment.** Constructing `nn.Conv1d` consumes RNG at init (kaiming_uniform_), which shifts the RNG state for every subsequent block's `qkvo_proj` random init and *breaks the step-0 byte-identity claim* across blocks 2..12. This is the same pattern 157-conv-ffn's `ConvFFN.__init__` uses (`models/conv_ffn.py:64-72`) and 156-moa's `moa_extra_kv` uses. The idea.md text mentions "Same trick as 157" but inconsistently specifies `nn.Conv1d(...)` in the design sketch — the plan.md must correct this to the raw-`Parameter` form.
  - **Verify step-0 bit-identity empirically.** `MinimalLLM(use_v_mix_conv=False)` vs `MinimalLLM(use_v_mix_conv=True)` at the same seed (42) MUST produce `max_abs_diff < 1e-6` on a 2×32 fp32 forward across all 12 blocks. Pin this as a self-check in `plan.md`'s "Step-0 byte-identity" section, mirroring the test pattern in 157's evidence.
  - **Apply-site ordering must be documented in the plan.** The conv sits *post-SDPA, post-reshape [B,T,H,D]→[B,T,d_model], pre-W_O projection*. The `MultiHeadAttention.forward` site is the same `attn_output` already used by the closed `out_op` family; the plan should cite the exact line number in `models/layers.py` and confirm no interaction with `use_attn_output_gate`, `use_attn_output_channel_gate`, `use_exclusive_self_attn`, `use_kda_channel_gate`, `use_head_gain`, `use_gated_attn`, or `out_op`.
  - **Padding = k//2 symmetric (causal+future).** Confirmed by the taste caveat — do NOT switch to causal padding mid-run; the attention sublayer has already integrated full causal context so the conv is allowed to look at both neighbors. Pin k=3 (spec pin).
  - **The 12L × 3 × 64 = 2,304 extra-params arithmetic is correct** (depthwise Conv1d weight shape = `[d_model=64, 1, k=3]` = 192 params/layer × 12 layers = 2,304; +0.25% of 0.94M). No re-count needed.

Verdict: **approve** → `needs-plan`, round reset to 1 so the code-implementer gets a fresh budget.
