# Taste log — 022 softpick-attention

## r1 — 2026-06-10 — verdict: accept
- **Sharp bet (passes the "one sentence" test).** "Tiny models with few heads waste the most attention mass on sinks → removing the forced-probability constraint frees head capacity → expect a small val-loss drop or a cleaner-activation null." Cites the *mechanism* (sink pathology in softmax with few heads), not just vibes.
- **Portfolio fit (diversifies).** Last 10 closed/active entries are mostly optimizers (001, 002, 003, 004, 005, 006, 011), positional (009, 013, 015, 016), or normalization (017). Softpick is a clean **attention-function lever** — distinct from `attn-sink` (closed, which *added* a sink token), `NSA / diff-attn / hybrid heads` (closed), and the adjacent attention-side 020 (forgetting) and 021 (V-residual). The portfolio is hungry for a softmax-replacement A/B.
- **Mechanism, not an HP.** 30 LoC, drop-in, no params added, no schedule, no init tuning, ε=1e-6 (negligible). A function swap — the cleanest kind of A/B.
- **Information value either way.** A null result is informative (sinks are benign at 0.94M / 6L — matches the 017-sub-ln-sandwich "expected null at 6L" pattern, but for a different axis). A win is a free, parameter-free, transferable architectural lever.
- **Step-0 caveat (carry, don't block).** Softpick(x) → 0 for x≤0; at init the QK/√d scores are ≈0, so output ≈ 0 attention — *not* softmax's small-nonzero start. The miner flagged this; the definition/recode gate should confirm numerically that step-0 loss/grads stay finite (ε in the denominator handles div-by-zero; exp(0)-1 = 0 → numerator exactly 0 → 0/ε = 0). If step-0 is effectively "no attention", the first dozens of steps are pre-softmax-with-cold-start, which is testable in the smoke build, not a taste problem.
- **Scope-clean.** Runs at tiny1m3m / seed 42, no HP sweep, no multi-seed claim, no tier mismatch on the bet (unlike 018 AdEMAMix, which the taste gate already rejected for that reason).
- **Risk noted, not penalized.** The original Softpick paper validates at 1.5B-7B. At 0.94M, the sink pathology may be a non-issue and the lever may null — but a clean null at this scale *is the result we want logged*, and the definition gate can confirm the swap is wired correctly so the null is interpretable.
