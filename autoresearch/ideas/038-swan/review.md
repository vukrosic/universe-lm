## r1 — 2026-06-11 — verdict: revise
- **Source verified.** arXiv:2412.13148 "SWAN: SGD with Normalization and Whitening Enables Stateless LLM Training" — Ma, Gong, Scetbon, Meeds (Dec 2024 / Feb 2025 v3). LLaMA 350M / 1.3B, ~2x token-to-perplexity vs Adam. Citation and authors are real and current. transfer-risk:low is consistent with the cited 1.3B scale.
- **Mechanism is structural, not HP.** Stateless gradient pre-processing (normalize → whiten) on matrix params — qualifies as a real optimizer-path swap (not a tuned constant). Step-0 is NOT bit-identical to the current baseline because baseline already routes matrix grads to Muon (Newton-Schulz orthogonalization), and SWAN replaces that with whitening — call this out explicitly in the idea so the code-reviewer doesn't flag the missing identity case.
- **Not a duplicate.** No SWAN / whitening / stateless lever in `autoresearch/closed.md`. Most adjacent closed optimizer rows are AdamW-family (002 cautious-AdamW null, 003 SOAP null, 006 schedule-free-AdamW null) or Muon variants (001 cautious-Muon null, 005 decoupled-qkv-Muon null, 015 Moonlight-Muon-RMS WIN). SWAN is distinct.
- **REVISE — pin the control explicitly.** Taste already flagged the carry-forward (taste.md:9) and the idea text never resolves it. The current baseline in `training/trainer.py:111-176` routes `param.ndim == 2 and 'token_embedding' not in name and 'norm' not in name` to Muon; everything else to AdamW. The meaningful A/B is therefore **SWAN-on-the-Muon-slot vs Muon-on-the-Muon-slot** (1-D / norm / embedding stay on AdamW in both arms). Add a `## Control` section that names this exactly, mirroring the wording used in 011-cautious-lion (Lion replaces Muon on the same 2-D slot).
- **REVISE — falsifiable pass/fail bar missing.** Frontmatter has no numeric target. Box noise at tiny1m3m is ~±0.01 val loss (ctrl gap from closed.md WIN rows is 0.0047–0.0175). Add a `## Pass bar` line: e.g. `Δ val ≤ -0.005 vs Muon-on-matrix ctrl at fixed step count (seed 42, tiny1m3m), passes only if Δ exceeds the within-session ctrl-gap`. The paper's "2x speedup" framing is iso-perplexity at variable tokens — that doesn't translate; this pipeline runs fixed steps, so the bar must be Δ val at fixed compute.
- **REVISE — LR / momentum / clipping unspecified.** The paper uses its own LR + momentum schedule on top of the whitened gradient. State which of (a) inherit Muon's `muon_lr=0.024, muon_momentum=0.95` from `configs/llm_config.py:465-466`, or (b) port the paper's defaults verbatim and cite the table. Without this the code-implementer has to guess and the result is non-reproducible.
- **REVISE — non-matrix routing must be named.** Idea says "keep scalar/norm parameters on the normal path" (idea.md:15) but doesn't say what "normal" is. Spell it out: 1-D scalars, `*.norm.weight`, and `token_embedding`/`emb_proj` stay on AdamW (exactly the existing routing); the SWAN swap touches only the 2-D Muon slot. This blocks the same misread that bit 005-decoupled-qkv-Muon.
- **OPTIONAL but recommended.** Add `## Identity case` noting that whitening matrix → I reduces SWAN to ordinary SGD on matrix params, not to the current Muon baseline. So a "step-0 identical to ctrl" check does not apply here; the reviewer/runner should compare end-of-training only.
